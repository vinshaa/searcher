Do not fill this in!       ==Automated design methods{{anchor|Design methods}}== Automated tools can aid in the creation of a Web service. For services using WSDL, it is possible to either automatically generate WSDL for existing classes (a bottom-up model) or to generate a class skeleton given existing WSDL (a top-down model). * A developer using a bottom-up model writes implementing classes first (in some programming language), and then uses a WSDL generating tool to expose methods from these classes as a Web service. This is simpler to develop but may be harder to maintain if the original classes are subject to frequent change.<ref>{{cite web | url = http://help.eclipse.org/help33/index.jsp?topic=/org.eclipse.jst.ws.doc.user/concepts/cwsbtmup.html | title = Help - Creating bottom-up Web services | publisher = [[Eclipse (software)|Eclipse]] | date = | accessdate = 2011-04-22}}</ref> * A developer using a top-down model writes the WSDL document first and then uses a code generating tool to produce the class skeleton, to be completed as necessary. This model is generally considered more difficult but can produce cleaner designs and is generally more resistant to change. As long as the message formats between sender and receiver do not change, changes in the sender and receiver themselves do not affect the Web service. The technique is also referred to as ''contract first'' since the WSDL (or contract between sender and receiver) is the starting point.<ref>{{cite web | url = http://help.eclipse.org/help33/index.jsp?topic=/org.eclipse.jst.ws.doc.user/concepts/cwstopdown.html | title = Help - Creating top-down Web services | publisher = [[Eclipse (software)|Eclipse]] | accessdate = 2011-04-22}}</ref> *  A developer using a Subset WSDL (SWSDL)<ref name=SWSDL>{{cite doi|10.1109/CloudCom.2014.149}}</ref> (i.e. a WSDL with the subset operation in the original WSDL) can perform web service testing and top down development.
