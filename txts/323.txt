Do not fill this in!       ==Language bindings for the FastCGI API== FastCGI can be implemented in any language that supports [[network socket]]s.  (Since "FastCGI is a protocol, not an implementation," it is not tightly bound to any language at all.) [[Application programming interface|API]]s exist<ref>[http://www.fastcgi.com/drupal/node/5 Application Libraries / Development Kits]</ref> for: * [[Ada (programming language)|Ada]]<ref>[http://forge.ada-ru.org/matreshka Matreshka]</ref> * [[Borland Delphi]]/[[Lazarus (IDE)|Lazarus]] [[FreePascal]]<ref>[http://extpascal.googlecode.com ExtPascal]</ref> * [[C (programming language)|C]] / [[C++]] * [[Chicken Scheme]] * [[Common Lisp]]<ref>[http://cliki.net/fastcgi How to use FastCGI from Common Lisp]</ref> * [[D (programming language)|D programming language]] * [[Eiffel (programming language)|Eiffel]]<ref>[http://goanna.origo.ethz.ch Goanna Eiffel]{{dead link|date=October 2014}}</ref> * [[Erlang (programming language)]] * [[Go (programming language)|Go]] * [[Guile Scheme]] * [[Haskell (programming language)|Haskell]] * [[HP BASIC for OpenVMS]] * [[Java (programming language)|Java]]<ref>[http://jfastcgi.sourceforge.net/ jFastCGI, a Java Servlet implementing FastCGI protocol]</ref> * [[Lua (programming language)|Lua]] * [[node.js]]<ref>[https://www.npmjs.org/package/node-fastcgi node-fastcgi npm package]</ref> * [[OCaml]] * [[Perl]]<ref> There are a number of FastCGI modules for Perl: [http://metacpan.org/module/FCGI FCGI] (a compiled module written in C), [http://metacpan.org/module/FCGI::Async FCGI::Async] (for asynchronous FastCGI applications), [http://metacpan.org/module/AnyEvent::FCGI AnyEvent::FCGI] (for [http://metacpan.org/module/AnyEvent AnyEvent]-based applications), [http://metacpan.org/module/FCGI::EV FCGI::EV] (for [http://metacpan.org/module/EV EV]-based applications), [http://metacpan.org/module/CGI::Fast CGI::Fast] (Perl [http://metacpan.org/module/CGI CGI]-like interface for FastCGI), [http://metacpan.org/module/FCGI::Client FCGI::Client] (a FastCGI client library), and [http://metacpan.org/module/Net::FastCGI Net::FastCGI] (constants and functions to build and parse FastCGI messages). </ref> * [[PHP]] (via php-fpm, or [[HipHop for PHP]]<ref>[http://www.hhvm.com/blog/1817/fastercgi-with-hhvm FasterCGI with HHVM]</ref>) * [[Python (programming language)|Python]] * [[Realbasic|REALbasic (REAL Studio)]]<ref>[http://www.realsoftware.com/web/ REAL Studio Web Edition, builds web apps called via FastCGI]</ref> * [[Ruby (programming language)|Ruby]] * [[Rust (programming language)|Rust]]<ref>[//github.com/danielksb/rust-fcgi]</ref> * [[SmallEiffel]] * [[Smalltalk]]: [[FasTalk]] and [[Dolphin Smalltalk]] * [[Tcl]] * [[WebDNA]] * [[Vala (programming language)|Vala]] (via C bindings)  Recent frameworks such as [[Ruby on Rails]], [[Catalyst (software)|Catalyst]], [[Django (web framework)|Django]], [[Kepler (software)|Kepler]] and [[Plack (software)|Plack]] allow use with either the embedded interpreters ([[mod_ruby]]<!-- underscore is correct, see article -->, [[mod_perl]], [[mod_python]] or [[Kepler (software)|mod_lua]], for example), or FastCGI.
